# 05. 외부 연동 및 자동화

## 1) 외부 연동 목록

### 1-1. 배차/운송
- 24시콜 API
  - 오더 등록/수정/조회/배차취소
  - 콜 상태 동기화(주기 작업)

### 1-2. 세금/정산
- Popbill
  - 매출세금계산서 발행/조회
  - webhook 수신으로 매입세금계산서 적재

### 1-3. 메시징/알림
- 알리고 SMS
- 알리고 알림톡(Kakao 템플릿)
- Slack Webhook(신규오더/취소/오류)

### 1-4. 기타
- 유니패스(통관/장치장 정보 조회)
- 카카오 선물하기 Biz(잔액/발송)
- 지도/경로
  - Naver Direction
  - Kakao Local Geocoding
  - Tmap 거리/시간/통행료

## 2) API 계층에서 제공하는 커스텀 엔드포인트

주요 업무 엔드포인트
- `/distance` 거리 조회
- `/sms` 문자 발송
- `/bills` 업체별 거래 집계
- `/unclaimed-bills` 미청구 주문 집계
- `/batch-billmonth` 주문 billMonth 일괄 지정/해제
- `/batch-finish` 주문 지급완료 일괄 처리
- `/send-kakao` 알림톡 발송
- `/unipass` 통관 조회
- `/driverinfo` 기사 과거이력 조회
- `/distance-price` 거리x톤수 평균 매입가 조회

세금/콜/분석
- `/tax/issue`, `/tax/list`, `/tax/detail`, `/tax/htdetail`, `/webhook`
- `/call/post`, `/call/match`
- `/dispatch-analytics/trend`, `/dispatch-analytics/share`

## 3) 정기 자동화(Cron)

### 3-1. 24시콜 상태 동기화
- 작업: `getCallInfo`
- 주기: 매 5분
- 내용: callOrderId가 있는 주문의 콜 상태/업데이트시각 갱신

### 3-2. 24시콜 오더 갱신
- 작업: `updateCallOrder`
- 주기: 매 3분
- 대상: 콜 상태 `접수` + 주문 상태 `requested/feeRequested`
- 내용: 콜 시스템 오더 정보 재전송/갱신 시도

### 3-3. 매입세금계산서 자동매칭
- 작업: `autoMatchIncomingTax`
- 주기: 매시 30분
- 내용: `pending` 계산서를 주문과 매칭, 성공 시 정산 단계 전환

### 3-4. 미처리 콜 연동 정리
- 작업: `cancelTodayOrders`
- 주기: 매일 23:50(KST)
- 내용: 접수중 상태 주문의 콜 연동 필드 초기화

## 4) 자동화 실패 시 운영 영향
- 콜 연동 실패: 배차 상태 동기화 지연, 수동 조회 필요
- 자동매칭 실패: review 큐 누적, 운영자 수동개입 증가
- 팝빌 실패: 청구 발행 지연, 수금 일정에 영향
- 메시징 실패: 기사/화주 커뮤니케이션 누락

## 5) 분석/리포팅 연동 규칙

### 5-1. 배차 추이 분석 메트릭
- `count`: 건수
- `revenue`: 매출 합계
- `profitRate`: 주문별 수익률 평균
- `companyCount`: 유니크 회사수

### 5-2. 필터 해석
- 담당자 필터는 회사 담당자(정/부) 기준으로 회사 집합을 만든 뒤,
- 실제 그래프 집계는 주문의 실배차 담당자(Order.manager) 기준으로 표시

## 6) 운영 관점 체크포인트
- 외부 연동 키/웹훅/비밀값이 코드에 직접 포함된 구간 존재
- 네트워크/벤더 장애 시 재시도·알림·복구 절차 표준화 필요
- webhook/배치 처리 모니터링 대시보드 부재
