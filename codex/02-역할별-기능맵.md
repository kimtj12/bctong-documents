# 02. 역할별 기능 맵

## 1) 역할 정의
- `admin`: 전체 운영 통합 관리자
- `partner`: 배차 매니저(운영 참여 + 일부 관리자 기능)
- `freelancer`: 영업 프리랜서(담당 회사 범위 중심)
- `user`: 고객사 사용자

참고
- 일반 사용자(`user`)는 `/admin` 경로 접근이 차단됨
- 화면 노출은 메뉴 권한 설정 + 사용자 권한 조합으로 제어됨

## 2) 역할별 핵심 목적
- `user`: 배차 요청/내역 추적, 거래명세·마일리지 확인, 자체 마스터(거래처/상차품) 유지
- `partner`: 배차 처리 및 기사 관리 보조
- `freelancer`: 담당 회사 영업/운영 실적 확인 및 관련 처리
- `admin`: 정산/청구/세금계산서/계정/정책 전체 책임

## 3) 메뉴 기능 맵
### A. 업체(`user`) 중심 메뉴
- 신규 배차 요청
- 배차내역
- 거래명세서
- 거래처 관리
- 상차품 관리
- 나의 마일리지
- 내 회사 관리
- 서식자료실

### B. 배차 관리(운영자)
- 배차상황실(일반 오더)
- 선착불 배차상황실

### C. 화주 정산 관리
- 업체별 거래내역
- 청구 내역 생성
- 청구 및 수금 내역
- 미수금 관리
- 매출 전자세금계산서

### D. 차주 관리
- 매입 전자세금계산서
- 지급 관리
- 기사 블랙리스트

### E. 관리자 메뉴
- 계약처 관리
- 상/하차지 관리
- 상차품 관리
- 평균매입가 조회
- 알림센터
- 마일리지 내역
- 상품권 설정
- 사용자 계정 관리

### F. 실험실/분석
- 기존 내역 분석
- 거리별 운임 단가
- 상하차지 거리 계산기
- 배차 추이 분석(코드 존재)

## 4) 역할별 실행 가능한 대표 작업
- `user`
  - 신규 오더 등록/수정(배차중 상태 한정)
  - 본인 회사 주문 이력 조회/엑셀
  - 본인 회사 거래명세 조회
  - 포인트 조회/출금 요청
- `partner`
  - 배차상황실에서 기사 매칭, 배차 취소, 정보 수정
  - 기사 블랙리스트 등록/수정
- `freelancer`
  - 담당 회사 범위의 주문/청구/미수 조회
  - 일부 관리자 메뉴 조회 및 제한적 실행
- `admin`
  - 매입세금계산서 매칭 승인/거절/재처리
  - 지급 확정(단건/일괄)
  - 청구 생성/삭제/발행/수금 등록
  - 외부 세금계산서 조회/문서 확인
  - 사용자 권한/소속 관리

## 5) 권한 관련 특이 규칙
- 회계 고급 권한(`hasAccountingPermission`)은 특정 사용자 ID에 하드코딩되어 있음(1, 3, 5, 87)
- 동일 메뉴라도 role에 따라 데이터 범위가 달라짐
  - 프리랜서는 담당 회사 목록 기준으로 필터링
  - 일반 사용자는 본인 회사 데이터만 조회
