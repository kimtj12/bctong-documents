# 03. 핵심 업무 플로우

## 1) 신규 배차 등록 플로우

### 1-1. 입력 항목
- 상차지/하차지 정보(명칭, 주소, 연락처, 좌표)
- 상/하차 희망일시(당상/내상/월상/기타 + 시간)
- 화물 정보(중량, 내용, 옵션: 인수증/왕복/혼적)
- 요청 차량(톤수/차종)
- 화주사(관리자일 때 선택)
- 첨부(화물 사진 필수 정책)
- 선택적으로 거리/시간/통행료 계산값

### 1-2. 등록 시 내부 처리
- 일반 오더: `status = requested`
- 선착불 오더: `status = feeRequested`
- 상차/하차 타입(today/tomorrow/monday/etc)을 실제 날짜로 변환
- 당일 하차(today)면 하차시간을 `바로당착`으로 보정
- 이미지 파일 업로드 또는 기존 상차품 이미지 재사용

### 1-3. 수정 가능 조건
- 기본적으로 배차중 상태에서 수정 중심
- 수정 시 변경 필드 요약을 알림(`notification`)으로 남김(오더 수정 추적)

## 2) 배차상황실(일반 오더) 플로우

### 2-1. 배차 매칭(핵심)
- 기사 정보 입력(이름/연락처/차량번호/차량정보)
- 상하차일시, 매입가(driverPrice), 매출가(price) 입력
- 저장 시 전이
  - `requested -> accounting`
  - 이미 진행중이면 현 상태 유지 후 정보 업데이트
- 매칭 시각(`matchDatetime`) 기록

### 2-2. 운영 보조 기능
- 과거 유사건 평균 매입가 조회(같은 권역/톤수/회사 기준)
- 24시콜 연동
  - 등록/수정/조회/배차취소 요청
  - 연동 해제(배차통의 call 연동 필드 초기화)
- 화주 변경, 단건 삭제, 카카오 공유, SMS 발송
- 유니패스 코드 기반 통관/장치장 정보 조회

### 2-3. 취소/롤백
- 배차 취소: 기사 정보 제거 + `status=requested`로 롤백
- 화물 취소: `status=canceled`

## 3) 선착불 배차상황실 플로우

### 3-1. 일반 오더와 차이
- 시작 상태: `feeRequested`
- 배차 입력 시 수수료(`fee`) 필수
- 저장 시
  - `status = feeFinished`
  - `price = driverPrice + fee` 자동 재산정

### 3-2. 취소 규칙
- 배차 취소 시 `status=feeRequested`로 롤백
- fee/price 등 선착불 관련 값 초기화

## 4) 기사 정산(지급) 플로우

### 4-1. 정산 진입
- 일반적으로 `accounting` 상태 주문이 정산 대상
- 진입 경로
  - 수기 입력(운영자)
  - 매입세금계산서 자동매칭(크론)
  - 매입세금계산서 수동매칭(운영자)

### 4-2. 정산 확정 입력
- 필수: 사업자번호, 운수사명, 은행, 예금주, 계좌번호, 계산서 발행일, 지급예정일
- 저장 시 `status=withdraw`
- 비관리자 사용자 주문이면 포인트 로그 +1000 생성

### 4-3. 지급 확정
- 단건 또는 일괄 처리로 `status=finished`, `driverFinalDate` 입력
- 지급 대상 리스트 엑셀(전체내역/은행대량이체 형식) 다운로드 지원

## 5) 매입세금계산서 처리 플로우

### 5-1. 수신
- Popbill webhook으로 매입 계산서 수신 시 `incoming-tax` 생성(`pending`)

### 5-2. 자동매칭(정시 배치)
- 조건: 주문 상태 `accounting`, 금액 일치, 차량번호 단서 일치
- 실패 사유 분류
  - `NO_ORDER`, `MULTIPLE_ORDERS`, `TAX_ISSUE`, `NO_BANK_INFO`
- 성공 시
  - 주문 `withdraw` 전환
  - 세금계산서와 주문 연결
  - 계좌/지급예정일 업데이트
  - 비관리자 유저 포인트 +1000
  - 기사 안내 알림톡 발송

### 5-3. 수동검증
- 운영자가 매칭/승인/거절/재수신 전환 가능
- 거절 시 사유 기록(`declineMemo`)

## 6) 화주 청구 플로우

### 6-1. 미청구 조회
- 청구월 미지정 주문(`billMonth is null`)을 회사별로 집계

### 6-2. 청구 생성
- 선택 주문으로 Invoice 생성
- 동시에 주문의 `billMonth` 일괄 설정
- 수기 청구 생성도 지원(`isManual=true`)

### 6-3. 청구 취소/삭제
- Invoice 삭제 시 연동된 주문 `billMonth`를 다시 null로 복원

## 7) 매출 세금계산서/수금 플로우

### 7-1. 발행
- 팝빌 발행: `/tax/issue`
- 수기(국세청 직접 발행 후 기록): `mgtKey="M"`로 표시
- 발행 결과를 Invoice에 저장
  - `mgtKey`, `ntsCode`, `taxPrice`, `taxDetail`, `paymentDueDate` 등

### 7-2. 수금
- 수금 로그(`invoice-log`)를 누적 기록
- Invoice의 `income`, `incomeDate`, `isPaid` 갱신
- `income == taxPrice`가 되면 완료 처리

### 7-3. 미수 관리
- `isPaid=false`이면서 `paymentDueDate` 경과 건을 미수로 분류
- 회사별 미수합계/초과일수 중심 관리

## 8) 마일리지 플로우
- 적립: 주문 정산 확정 시(조건부) 포인트로그 +1000
- 차감: 출금/운영 처리 시 포인트로그 음수 생성
- 사용자 총 포인트는 포인트로그 생성 후 자동 반영

## 9) 운영 알림 플로우
- 가격 변경, 오더 수정, 오더 취소, 수수료 수정 등 주요 이벤트를 `notification`에 누적
- 목적: 운영 이력 추적, 사후 커뮤니케이션 근거 확보
