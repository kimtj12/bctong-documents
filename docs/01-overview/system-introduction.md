# 서비스 소개

## 배차통이란?

**배차통**은 화물 운송 중개 플랫폼입니다. 화물을 보내고 싶은 업체(화주)와 화물을 운송하는 기사(차주)를 연결하여, 효율적인 물류 운송을 지원합니다.

### 서비스 명칭
- 정식 명칭: 배차통
- 영문명: BCTong (Baecha Tong)
- 운영사: 바른물류연구소

---

## 서비스 목적과 가치

### 해결하는 문제

**기존 방식의 문제점:**
1. 화주가 기사를 직접 찾아야 함 (전화, 인맥)
2. 가격 협상이 매번 번거로움
3. 운송 현황 파악이 어려움
4. 정산/청구 업무가 수작업으로 진행
5. 세금계산서 처리에 많은 시간 소요

**배차통의 해결책:**
1. 플랫폼에서 적합한 기사를 자동/수동 배정
2. 거리 기반 운임 자동 계산
3. 실시간 배차 상태 확인
4. 청구서/정산 자동화
5. 전자세금계산서 자동 발행

### 핵심 가치

| 가치 | 설명 |
|------|------|
| **효율성** | 배차 요청부터 정산까지 원스톱 처리 |
| **투명성** | 운임 계산 기준이 명확, 실시간 상태 확인 |
| **자동화** | 반복 업무 자동화로 시간 절약 |
| **신뢰성** | 기사 평가/관리 시스템 |

## 주요 사용자

### 1. 화주 (화물 의뢰 업체)

화물 운송을 의뢰하는 업체입니다.

**주요 니즈:**
- 빠르고 안전한 화물 운송
- 합리적인 운송 비용
- 간편한 배차 요청
- 정확한 청구서/거래명세서

**대표 업종:**
- 수출입 물류회사
- 제조업체
- 유통업체
- 이커머스 업체

### 2. 차주 (화물 운송 기사)

실제 화물을 운송하는 기사입니다.

**주요 니즈:**
- 안정적인 배차 물량 확보
- 정확하고 빠른 운임 정산
- 간편한 운송 정보 확인

**차량 종류:**
- 다마스/라보 (0.5톤)
- 1톤, 1.4톤, 2.5톤
- 3.5톤, 5톤
- 11톤, 18톤, 25톤

### 3. 배차 담당자 (운영자)

배차통을 운영하고 배차를 관리하는 직원입니다.

**주요 업무:**
- 배차 요청 접수 및 기사 배정
- 운송 진행 상황 관리
- 화주/기사 정산 처리
- 고객 응대

## 서비스 범위

### 지원 운송 유형

| 유형 | 설명 |
|------|------|
| 일반 화물 | 일반적인 화물 운송 |
| 왕복 운송 | 출발지로 다시 돌아오는 운송 |
| 혼적 운송 | 여러 화물을 함께 운송 |
| 수출입 화물 | 항만/공항 관련 운송 |

### 지원 차량

| 분류 | 차량 종류 |
|------|----------|
| 소형 | 다마스, 라보, 1톤 |
| 중형 | 1.4톤, 2.5톤, 3.5톤 |
| 대형 | 5톤, 5톤 플러스, 5톤 플축 |
| 특대형 | 11톤, 18톤, 25톤 |

### 차량 형태

- **카고**: 일반 적재함
- **윙바디**: 측면 개방형
- **탑차**: 밀폐형 적재함
- **냉동/냉장**: 온도 유지 차량
- **리프트**: 지게차 탑재

## 비즈니스 모델

```
화주 ──(운송료)──> 배차통 ──(기사 운임)──> 차주
                    │
                    └── (수수료) = 운송료 - 기사 운임
```

### 수익 구조
1. **운송 중개 수수료**: 화주 운송료와 기사 운임의 차액
2. **부가 서비스**: 빠른 지급 서비스 수수료 (1.5%)

### 정산 주기
- **화주 청구**: 월 단위 합산 청구
- **기사 지급**: 건별 또는 주 단위 정산

## 경쟁 우위

### vs 일반 화물 중개

| 항목 | 일반 중개 | 배차통 |
|------|----------|--------|
| 배차 방식 | 전화/문자 | 웹 시스템 |
| 운임 계산 | 수동 협상 | 자동 계산 |
| 정산 | 수작업 | 자동화 |
| 세금계산서 | 수동 발행 | 자동 발행 |
| 운송 추적 | 전화 확인 | 실시간 상태 |

### 핵심 차별점

1. **24시콜 연동**: 외부 배차 플랫폼과 자동 연동으로 기사 매칭 확대
2. **세금계산서 자동화**: Popbill 연동으로 발행/수신 자동 처리
3. **거리 기반 운임**: 네이버 지도 API 연동으로 정확한 거리/요금 계산
4. **실시간 알림**: Slack, 카카오톡으로 즉시 알림

## 시스템 구성

배차통은 두 가지 주요 시스템으로 구성됩니다:

### 웹 대시보드 (web)
- 화주와 관리자가 사용하는 웹 인터페이스
- 배차 요청, 내역 조회, 정산 관리 등

### API 서버 (api)
- 데이터 처리와 외부 시스템 연동 담당
- 모든 비즈니스 로직 처리

> 상세한 시스템 구조는 [시스템 구조](./architecture.md) 문서를 참조하세요.
