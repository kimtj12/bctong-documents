# 현재 시스템 한계점

v1 운영 중 발견된 개선 필요 영역을 정리합니다.

---

## 1. 기능적 한계

### 1.1 배차 관리

| 한계점 | 상세 설명 | 영향 |
|--------|----------|------|
| **수동 기사 배정** | 24시콜 외에는 관리자가 직접 기사를 찾아 배정해야 함 | 배차 시간 증가 |
| **실시간 추적 부재** | 운송 중 위치 추적 기능 없음 | 화주 문의 증가 |
| **배차 자동화 부족** | 반복 배차의 자동 생성 기능 없음 | 반복 작업 |
| **견적 시스템 미흡** | 즉시 견적 제공 어려움 | 신규 고객 유치 어려움 |

### 1.2 정산/청구

| 한계점 | 상세 설명 | 영향 |
|--------|----------|------|
| **수동 청구 프로세스** | 미청구 조회 → 청구월 지정이 수작업 | 정산 지연 |
| **세금계산서 매칭** | 자동 매칭 실패 시 수동 확인 필요 | 업무 부담 |
| **부분 입금 추적** | 여러 번 입금 시 추적이 복잡 | 정산 오류 가능 |
| **미수금 알림 없음** | 미수금 발생 시 자동 알림 없음 | 연체 관리 어려움 |

### 1.3 기사 관리

| 한계점 | 상세 설명 | 영향 |
|--------|----------|------|
| **기사 풀 관리 미흡** | 자체 기사 네트워크 관리 기능 부족 | 기사 확보 어려움 |
| **평가 시스템 없음** | 기사 서비스 품질 평가 체계 없음 | 품질 관리 어려움 |
| **블랙리스트만** | 화이트리스트는 있으나 적극 활용 안됨 | 우수 기사 관리 부족 |

### 1.4 알림/커뮤니케이션

| 한계점 | 상세 설명 | 영향 |
|--------|----------|------|
| **단방향 알림** | 화주/기사에게 알림만 발송, 응답 수신 불가 | 소통 제한 |
| **알림 커스터마이징 불가** | 알림 내용/시점 사용자 설정 불가 | 불필요한 알림 |
| **푸시 알림 미흡** | 웹 푸시만 지원, 앱 푸시 없음 | 알림 도달률 낮음 |

## 2. 사용성 이슈

### 2.1 화면/UI

| 이슈 | 상세 설명 |
|------|----------|
| **복잡한 배차 입력** | 많은 필드를 한 화면에서 입력해야 함 |
| **모바일 최적화 부족** | 반응형이나 모바일 전용 UI 아님 |
| **검색 기능 제한** | 고급 검색/필터 옵션 부족 |
| **대시보드 커스터마이징 불가** | 사용자별 대시보드 설정 불가 |

### 2.2 업무 흐름

| 이슈 | 상세 설명 |
|------|----------|
| **여러 단계의 클릭** | 간단한 작업에도 여러 화면 이동 필요 |
| **일괄 처리 제한** | 일부 기능만 일괄 처리 지원 |
| **키보드 단축키 없음** | 모든 작업에 마우스 필요 |
| **실행취소 없음** | 잘못된 작업 되돌리기 어려움 |

### 2.3 접근성

| 이슈 | 상세 설명 |
|------|----------|
| **모바일 앱 없음** | 현장에서 스마트폰 접근 어려움 |
| **오프라인 지원 없음** | 인터넷 연결 필수 |
| **다국어 미지원** | 한국어만 지원 |

## 3. 데이터 구조 한계

### 3.1 Order 엔티티

| 이슈 | 상세 설명 |
|------|----------|
| **필드 수 과다** | 70개 이상의 필드로 복잡 |
| **혼재된 정보** | 기사, 정산, 상태 등이 한 테이블에 |
| **확장 어려움** | 새 필드 추가 시 영향 범위 큼 |

### 3.2 관계 구조

| 이슈 | 상세 설명 |
|------|----------|
| **다대다 관계 부재** | 거래처-담당자 관계가 유연하지 않음 |
| **이력 관리 미흡** | 변경 이력 추적 기능 제한적 |
| **버전 관리 없음** | 데이터 버전 관리 없음 |

---

## 4. 확장성 문제

### 4.1 시스템 확장

| 문제 | 상세 설명 |
|------|----------|
| **단일 서버** | 부하 분산 구조 아님 |
| **동기식 처리** | 대용량 처리 시 병목 |
| **캐싱 미흡** | 반복 조회 성능 저하 |

### 4.2 비즈니스 확장

| 문제 | 상세 설명 |
|------|----------|
| **단일 운송 모드** | 육상 화물만 지원 |
| **국내 전용** | 국제 운송 지원 불가 |
| **고정 수수료 구조** | 유연한 가격 정책 어려움 |

### 4.3 통합 확장

| 문제 | 상세 설명 |
|------|----------|
| **제한된 API** | 외부 시스템 통합 API 부족 |
| **웹훅 미지원** | 외부에서 이벤트 수신 불가 |
| **SSO 미지원** | 기업 인증 시스템 연동 불가 |

## 5. 운영 한계

### 5.1 모니터링

| 한계점 | 상세 설명 |
|--------|----------|
| **실시간 대시보드 없음** | 운영 현황 한눈에 파악 어려움 |
| **성능 모니터링 미흡** | 시스템 부하 추적 제한 |
| **이상 탐지 없음** | 비정상 패턴 자동 감지 불가 |

### 5.2 리포팅

| 한계점 | 상세 설명 |
|--------|----------|
| **정형 리포트만** | 미리 정의된 리포트만 제공 |
| **커스텀 리포트 불가** | 사용자 정의 리포트 생성 불가 |
| **예측 분석 없음** | 과거 데이터 기반 예측 불가 |

### 5.3 관리

| 한계점 | 상세 설명 |
|--------|----------|
| **권한 관리 경직** | 4단계 권한만 지원 |
| **감사 로그 미흡** | 상세 활동 추적 어려움 |
| **백업/복구 수동** | 자동 백업 체계 미흡 |

## 우선순위 제안

### 높음 (즉시 개선 필요)

1. 모바일 앱 또는 모바일 최적화
2. 실시간 운송 추적
3. 자동 청구/정산 프로세스
4. 기사 풀 관리 시스템

### 중간 (중기 개선)

5. 알림 시스템 고도화
6. 리포팅/분석 강화
7. UI/UX 개선
8. API 확장

### 낮음 (장기 개선)

9. 다국어 지원
10. 국제 운송 지원
11. AI 기반 예측
12. 고급 자동화
