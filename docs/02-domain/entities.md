# 핵심 데이터 모델

배차통에서 관리하는 주요 데이터를 설명합니다.

---

## 개요

배차통은 다음 핵심 데이터를 관리합니다:

| 데이터 | 설명 | 핵심 역할 |
|--------|------|----------|
| **배차 (Order)** | 화물 운송 건 | 시스템의 중심 |
| **거래처 (Company)** | 화주 업체 | 고객 관리 |
| **상/하차지 (Partner)** | 출발/도착 장소 | 위치 관리 |
| **사용자 (User)** | 시스템 사용자 | 접근 관리 |
| **기사 (Driver)** | 운송 기사 | 품질 관리 |
| **청구서 (Invoice)** | 월별 청구 | 정산 관리 |
| **단가표 (Price)** | 구간별 운임 | 가격 관리 |
| **포인트 (PointLog)** | 마일리지 | 혜택 관리 |

---

## 1. 배차 (Order)

배차통의 **가장 핵심적인 데이터**입니다. 하나의 배차는 한 건의 화물 운송을 의미합니다.

### 기본 정보

| 정보 | 설명 | 예시 |
|------|------|------|
| 배차 번호 | 고유 식별 번호 | 12345 |
| 상태 | 현재 처리 단계 | 배차요청, 정산중, 완료 |
| 생성일 | 배차 요청일 | 2024-01-15 |

### 출발 정보 (상차지)

| 정보 | 설명 | 예시 |
|------|------|------|
| 출발지명 | 상차지 이름 | OO물류센터 |
| 출발 주소 | 상세 주소 | 인천시 중구 항동7가 |
| 출발 연락처 | 현장 연락처 | 010-1234-5678 |
| 상차 일시 | 상차 예정 시간 | 2024-01-15 09:00 |
| 상차 유형 | 당상/내상/월상 | 당상 (당일 상차) |

### 도착 정보 (하차지)

| 정보 | 설명 | 예시 |
|------|------|------|
| 도착지명 | 하차지 이름 | XX창고 |
| 도착 주소 | 상세 주소 | 서울시 강남구 역삼동 |
| 도착 연락처 | 현장 연락처 | 010-9876-5432 |
| 하차 일시 | 하차 예정 시간 | 2024-01-15 14:00 |
| 하차 유형 | 당착/내착 | 당착 (당일 하차) |

### 화물 정보

| 정보 | 설명 | 예시 |
|------|------|------|

### 차량 정보

| 차량 톤수 | 적재 용량 | 5톤 |
| 상차 방법 | 적재 방식 | 지게차 |
| 하차 방법 | 하역 방식 | 수작업 |

### 기사 정보

## 데이터 관계 요약

```
거래처 (Company)
├── 여러 배차 (Order) 의뢰
├── 여러 상/하차지 (Partner) 보유
├── 여러 단가표 (Price) 보유
├── 여러 청구서 (Invoice) 수신
└── 여러 사용자 (User) 소속

배차 (Order)
├── 하나의 거래처에 속함
├── 하나의 담당자가 관리
├── 하나의 수입세금계산서와 연결
└── 포인트 기록과 연결

사용자 (User)
├── 하나의 거래처에 소속
└── 여러 포인트 기록 보유
```
